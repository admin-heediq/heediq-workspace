# plans/

One `wip-*.md` file per in-flight branch. Created when an implementation session pauses mid-branch so
the next session (Step 0a) can resume cleanly.

## `wip-<branch>.md` should capture
- **Branch**: `<type>/<TICKET>-<desc>`
- **Goal**: one-line summary of what this branch does
- **State**: which Step we're on; which sub-steps are done/confirmed
- **Next**: the immediate next action
- **Open questions / risks**: anything unresolved

Delete the WIP file when the branch is merged.
